GAME Scheduler — what’s next.

A quarterly course scheduler currently tuned to SCAD’s Game department. The faculty, rooms, and offerings you see in the demo workbook are Eric Allen’s real data — treat them as sample input, not a commitment.

The tool is Excel-native: your workbook is the single source of truth, edits round-trip, and every export writes a dated backup. This page is the honest answer to “can we add our program / campus / faculty?”

Maintained by Tim Lindsey · tlindsey@scad.edu · Solo-built with occasional AI pair-programming sessions.

How do I add my program?

Short answer: today the tool ships with GAME / ITGM / MOME course catalogs and seven rooms in FORTY5 (Atlanta). Swapping in your program is a deliberate onboarding pass, not a copy-paste — we’d scrape your catalog, define the specialization tags that drive affinity, pick a color palette, and do a first solve on your real data together.

Rough effort per program: 1–2 weeks of my time for a well-scoped single-department pilot; longer if you want multiple departments or cross-campus scheduling. See “Scale beyond GAME” below for the specifics.

If you’re curious, not committed: the hosted demo is fine to poke at. Export a backup whenever you make changes — your workbook is yours, and nothing you do here affects anyone else’s file.

What’s live on the hosted build today.

  • Excel as source of truthYour workbook holds everything. Hidden _data_* sheets store state; visible sheets are printouts.
  • Three solve modes + Tune gearAffinity-first, Time-pref-first, Balanced. Live progress stream with a custom weight tuner.
  • Live validationStructured Data Issues panel calls out bad rows with sheet / row / reason. Fix in Excel, reload.
  • Auto-backup on exportEvery export drops a timestamped copy in .backups/. Ten most recent kept automatically.
  • Responsive across 3 breakpointsDesktop, tablet, phone. Schedule grid stays readable; mobile gets a single-column tap-to-place flow.
  • Mobile read-only viewerPublished schedules render at any size. Useful for handing a phone to a colleague mid-meeting.
  • Per-browser persistenceEdit professors, upload portraits, swap rooms — everything survives reload via localStorage.
  • Cloudflare-gated hostingAccess policy limits @scad.edu addresses; login is a one-time emailed PIN.

Workbook trust (Phase 4)

First-run empty state

3–4 days

A chair opening the tool with no workbook today sees a cold start. We’d add a “start from template” flow: pick a department shape, download a seeded .xlsx, land on a usable (if empty) schedule. Blocks serious onboarding for any new chair.

Formula-driven printout sheets

3–5 days

The human-readable sheets in your workbook are currently static snapshots. We’d rewrite them as formulas over the _data_* tables so edits to the printout round-trip into the app on reload. Makes “live in Excel, visit the app to solve” a real workflow.

Copy-cell-reference shortcut

0.5 day

Today Data Issues tells you the bad cell (e.g. _data_offerings!C3). One click should put that on your clipboard so you can paste into Excel’s Name Box and land exactly there. Tiny, high-leverage. 80% of the benefit of a full in-app fix modal, 5% of the cost.

Rescue flow for unreadable workbooks

1–2 days

If the hidden sheets get corrupted, today you’re stuck. We’d detect the failure, offer to restore from the latest local backup or the localStorage snapshot, and explain what happened. Cheap insurance against the one scenario that would burn trust.

Admin ergonomics

Portrait export / import

1–2 days

Right now portraits live in your browser only. We’d embed them in the workbook so they travel with your file and appear for anyone you share it with.

Print-ready schedule view

1–2 days

Browser-print CSS that produces a one-page grid suitable for handing to a dean or pinning in a hallway. No more Excel-then-screenshot.

Scale beyond GAME

Load another college (FASH, ADVE, SFA…)

1–2 weeks per college

Scrape the program’s catalog, define specialization tags that drive affinity matching, pick a department color palette, and run the solver against a real quarter of their data. First college is research-heavy; subsequent colleges get faster as the pattern solidifies. This is the “yes, we can add your program” answer.

Multi-campus support

~1 week

SCAD runs Savannah, Atlanta, and Lacoste. Today rooms are implicitly Atlanta. We’d add a campus attribute on rooms and professors, a campus filter in the roster, and a rule that a professor can’t teach two campuses in the same slot. Cross-campus solves become a mode.

Multi-department in one workbook

3–5 days

The solver already handles mixed departments. The UI currently assumes a single department leads the roster. Small lift to add a dept filter and per-dept sub-views without breaking the single-schedule output.

Chair-to-chair sharing

Template / starter-pack export

3–5 days

Export a de-personalized version of your workbook (rooms, course structure, preferences; no assignments) so another chair can start from your shape without inheriting your data.

Published snapshot library

~1 week

The mobile viewer is already scaffolded for read-only schedules. We’d finish the publish side: mark a solve as official, save it to a shareable index, let colleagues browse past quarters without needing the full app.

Prior-quarter diff view

~3 days

“What changed from Fall 2025?” Side-by-side of two published schedules, highlighting added / moved / dropped courses and prof reassignments. Depends on the snapshot library landing first.

UX / polish

Keyboard shortcuts

~2 days

Grid navigation (arrow keys to move between slots), / to search, ⌘S for export. The power-user affordance a chair grows into after a quarter of use.

Onboarding tour

3–4 days

First-load tooltip walkthrough: “this is your roster, here’s the grid, this button solves.” Dismissible, re-openable from About. Blocked on the first-run empty state landing first.

Undo / redo

~1 week

Ctrl-Z over placements, removals, and roster edits. Requires a small state-machine refactor but a big confidence gain for new users.

Horizon

Deferred / Research

Worth doing if a real user asks for them. Not scoped yet.

Solver sophistication

Section pairing / co-requisite timing

~1 week

“Section A meets right before Section B, same room.” Useful for labs paired to lectures, studios tied to crits. Solver constraint exists in theory; needs a UI affordance to author them.

Multi-quarter solve (annual planning)

2–3 weeks

Instead of one quarter at a time, plan a full academic year with rotating offerings and professor sabbatical windows. Large scope, new UI surface.

Historical enrollment fit

Research spike

Use past enrollment data to suggest section counts or predict over-subscription. Requires a data pipeline that doesn’t exist yet and a trust conversation with whoever owns enrollment data.

Custom constraint authoring

Research spike

Let chairs write their own hard and soft rules without code. Interesting but philosophically risky — most “custom rules” requests are better served by a setting we’d add anyway.

Cross-chair

Cross-chair comparison

Research spike

“How did the other GAME chair schedule this last year?” Requires institutional buy-in on what’s shareable and a story for permissions that doesn’t exist today.

Institutional hosting

Research spike

Today the app runs on a personal Fly.io account with Cloudflare Access gated to @scad.edu. Real institutional adoption would want IT-owned hosting, SSO, and an audit trail. Conversation precedes implementation.
